# 📘 Шпаргалка Директора: Алгоритм передачи контекста

## 🎯 Главная цель
**Директор всегда должен понимать, что было сделано ранее, чтобы принять правильное решение о продолжении или завершении работы.**

---

## ⚡ Краткая схема (3 шага)

```
1. ПОЛУЧЕНИЕ ЗАДАЧИ → Записать в память ✅
2. ЗАВЕРШЕНИЕ АГЕНТА → Записать результат ✅
3. ПРОВЕРКА → Прочитать из памяти → Решение ✅
```

---

## 📝 Шаг 1: При получении задания

### Что происходит:
Пользователь говорит: *"Сделай X"*

### Что записывается в `project.json`:
```json
{
  "task_history": [
    {
      "id": 1,
      "timestamp": "2025-01-15T10:30:00",
      "task": "Сделай X",
      "status": "in_progress"
    }
  ],
  "current_task": {
    "id": 1,
    "task": "Сделай X",
    "status": "in_progress"
  }
}
```

### Код в Conductor:
```python
await self._log_task_assignment(user_message)
```

---

## 📝 Шаг 2: После работы агента

### Что происходит:
Агент выполнил задачу и вернул отчёт

### Что записывается в `project.json`:
```json
{
  "completed_tasks": [
    {
      "id": 1,
      "task_id": 1,
      "status": "completed",
      "summary": "X выполнен",
      "files_created": ["file.py"],
      "errors": []
    }
  ],
  "current_task": {
    "status": "completed",
    "result_summary": "X выполнен"
  }
}
```

### Код в Conductor:
```python
if event.get("type") == "agent_done":
    await self._log_agent_completion(event)
```

---

## 📝 Шаг 3: Принятие решения

### Что делает Директор:
1. **Читает** память проекта
2. **Анализирует** что сделано
3. **Решает** что делать дальше

### Код в Conductor:
```python
memory_context = await self._get_project_context()
task_history = memory_context.get("task_history", [])
completed_tasks = memory_context.get("completed_tasks", [])
```

---

## 🤔 Таблица решений Директора

| Что видит Директор | Решение | Действие |
|-------------------|---------|----------|
| ✅ Все задачи выполнены | **YES** → Завершить | `action: "final"` |
| ⚠️ Есть ошибки | **NO** → Исправить | `action: "delegate"` + коррекция |
| ❓ Не хватает данных | **WAIT** → Спросить | `action: "ask_user"` |
| 🔁 Часть выполнена | **CONTINUE** → Далее | `action: "delegate"` + следующий шаг |

## 📊 Пример диалога

### Сценарий: Создание API

**Пользователь:** *"Создай REST API для задач"*

---

**🔄 Итерация 1:**

1. **Запись задания:**
   ```json
   {"current_task": {"task": "Создай REST API", "status": "in_progress"}}
   ```

2. **Делегирование:** coder → создать models.py
   
3. **Запись результата:**
   ```json
   {"completed_tasks": [{"summary": "models.py создан", "files_created": ["models.py"]}]}
   ```

4. **Решение Директора:** 
   - Видит: models.py готов, но нет routes
   - Решение: **CONTINUE** → делегировать создание routes

---

**🔄 Итерация 2:**

1. **Делегирование:** coder → создать routes.py

2. **Запись результата:**
   ```json
   {"completed_tasks": [..., {"summary": "routes.py создан", "files_created": ["routes.py"]}]}
   ```

3. **Решение Директора:**
   - Видит: models.py ✓, routes.py ✓
   - Решение: **YES** → завершить

---

## ✅ Чек-лист проверки перед завершением

Перед тем как сказать `"final"`, Директор проверяет:

- [ ] Все задачи из `task_history` имеют статус `completed`
- [ ] Нет ошибок в `completed_tasks[].errors`
- [ ] Все необходимые файлы созданы (`files_created`)
- [ ] Пользователь получил то, что просил

Если хотя бы один пункт **НЕТ** → продолжать работу!

---

## 🚨 Типичные ошибки

### ❌ Ошибка 1: Забыли записать результат
**Симптом:** Директор не видит что сделано  
**Решение:** Всегда вызывать `_log_agent_completion()` после `agent_done`

### ❌ Ошибка 2: Не читаем память перед решением
**Симптом:** Директор принимает решения без контекста  
**Решение:** Всегда вызывать `_get_project_context()` перед анализом

### ❌ Ошибка 3: Завершаем слишком рано
**Симптом:** Часть задач не выполнена  
**Решение:** Проверять чек-лист перед `action: "final"`

---

## 💡 Советы Директору

1. **Всегда читай память** перед принятием решения
2. **Сравнивай** task_history с completed_tasks
3. **Проверяй** наличие ошибок в результатах
4. **Уточняй** у пользователя если не уверен
5. **Завершай** только когда всё действительно готово

## 📄 Структура project.json (полная)

```json
{
  "task_history": [],        // Все полученные задачи
  "current_task": {},        // Текущая активная задача
  "completed_tasks": [],     // Все завершённые задачи
  "history": []              // Общая хронология событий
}
```

---

## 🎯 Ключевой принцип

> **Нет записи в память = задача не существовала**  
> **Нет чтения из памяти = решение без оснований**

Всегда записывай! Всегда читай!
