### 1. WinError 183 — зацикливание при записи
**Проблема:** `temp_file.rename()` не работает на Windows при существующем файле  
**Решение:** Использовать `os.replace()` вместо `rename()`  
**Статус:** ✅ Исправлено в `_log_task_assignment()` и `_log_agent_completion()`

### 2. Пользователь не видит сообщений в чате
**Проблема:** Сообщения отображаются как системные, без деталей  
**Решение:** Улучшены обработчики событий в `gui/app.py`:
- `_on_delegated()` — показывает роль и задачу с форматированием
- `_on_tool_call()` — показывает имя инструмента и параметры
- `_on_agent_done()` — показывает summary, файлы, ошибки

**Статус:** ✅ Исправлено

## 🟠 Приоритет P1 — Срочные улучшения (2-4 часа)

### 1. Проверка выполнения задач директором
**Проблема:** Director не проверяет, была ли задача реально выполнена, и продолжает делегировать  
**Решение:** Добавить проверку перед новым делегированием:

```python
# В _analyze_and_decide() после получения контекста
completed_tasks = memory_context.get("completed_tasks", [])
current_task = memory_context.get("current_task", {})

# Проверяем, есть ли успешное выполнение
if completed_tasks:
    last_task = completed_tasks[-1]
    if last_task["status"] == "completed" and last_task["task_id"] == current_task.get("id"):
        # Задача уже выполнена — переходим к финалу
        return FinalAction(result="Задача выполнена")
```

**Время:** 1 час  
**Влияние:** Критично — предотвращает зацикливание

### 2. Лимит неудачных попыток
**Проблема:** Цикл продолжается даже после нескольких неудач  
**Решение:** Считать количество failed задач и прерывать после 3:

```python
# В process_request() перед циклом
failed_count = sum(1 for t in completed_tasks if t["status"] == "failed")
if failed_count >= 3:
    yield {"type": "error", "message": "Превышен лимит неудачных попыток (3)"}
    return
```

**Время:** 30 минут  
**Влияние:** Высокое — экономит токены и время

### 3. Валидация названий инструментов
**Проблема:** LLM выдумывает названия (`file_write` вместо `write_file`)  
**Решение:** Добавить авто-коррекцию в Worker:

```python
# В worker.py перед вызовом LLM
available_tools = self.tool_registry.list_tools()
tool_names = [t.name for t in available_tools]

# Добавляем в system prompt подсказку
system_prompt += f"\n\nДоступные инструменты: {', '.join(tool_names)}"
system_prompt += "\nИспользуй ТОЛЬКО эти названия инструментов."
```

**Время:** 1 час  
**Влияние:** Высокое — улучшает выполнение задач

## 🟡 Приоритет P2 — Упрощение архитектуры (4-6 часов)

### 1. Объединить state.json + project.json
**Проблема:** Дублирование состояния в двух файлах  
**Решение:** Хранить всё в `project.json`:

```python
# Вместо state.json использовать project.json
data = {
    "state": {
        "stage": "executing",
        "iteration": 5,
        "active_role": "coder",
    },
    "task_history": [...],
    "completed_tasks": [...],
    "current_task": {...},
    "history": [...]
}
```

**Время:** 3 часа  
**Влияние:** Среднее — упрощает код, уменьшает bugs

### 2. Удалить protocol.py (избыточный)
**Проблема:** 200+ строк дублирования структур  
**Решение:** Использовать простые dict/dataclass напрямую в conductor.py  
**Время:** 2 часа  
**Влияние:** Среднее — уменьшает codebase

### 3. Кэширование контекста
**Проблема:** Чтение `project.json` на каждой итерации  
**Решение:** Кэшировать в памяти Conductor:

```python
class Conductor:
    def __init__(self):
        self._context_cache = None
        self._cache_timestamp = None
    
    async def _get_project_context(self):
        # Обновлять кэш раз в 30 секунд
        if self._context_cache and (datetime.now() - self._cache_timestamp).seconds < 30:
            return self._context_cache
        
        # Чтение из файла...
        self._context_cache = data
        self._cache_timestamp = datetime.now()
        return data
```

**Время:** 1 час  
**Влияние:** Низкое — небольшая оптимизация

## 🟢 Приоритет P3 — Надёжность (2-3 часа)

### 1. Таймауты на LLM-вызовы
**Проблема:** Бесконечное ожидание ответа от LM Studio  
**Решение:** Добавить timeout в `lm_client.py`:

```python
response = await asyncio.wait_for(
    self.client.chat_completion(...),
    timeout=60.0  # 1 минута
)
```

**Время:** 30 минут  
**Влияние:** Среднее — предотвращает зависания

### 2. Расширенное логирование
**Проблема:** Трудно отлаживать зацикливание  
**Решение:** Добавить логи ключевых решений:

```python
logger.info(f"ITERATION {self.iteration}: action={action.action_type}, "
            f"completed_tasks={len(completed_tasks)}, "
            f"failed_count={failed_count}")
```

**Время:** 1 час  
**Влияние:** Среднее — упрощает debug

### 3. Graceful shutdown
**Проблема:** Резкая остановка может повредить файлы  
**Решение:** Корректно завершать текущую операцию перед выходом  
**Время:** 1.5 часа  
**Влияние:** Низкое — улучшает UX
