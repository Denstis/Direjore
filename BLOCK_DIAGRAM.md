# Блок-схема: От получения задачи до финиша

## Визуальная схема процесса (Mermaid)

```mermaid
flowchart TD
    Start([📥 Получение задачи от пользователя]) --> Step1
    
    subgraph Step1["ШАГ 1: Запись в память проекта"]
        direction TB
        A1[Создать запись в task_history] --> A2[Установить current_task]
        A2 --> A3[Статус: in_progress]
        A3 --> A4[Сохранить timestamp и ID]
        A4 --> A5[💾 Записать в project.json]
    end
    
    Step1 --> Step2
    
    subgraph Step2["🤖 Работа агентов"]
        direction TB
        B1[Conductor распределяет задачу] --> B2[Агенты выполняют работу]
        B2 --> B3[Получение результатов]
        B3 --> B4{Есть ошибки?}
        B4 -->|Да| B5[Зафиксировать ошибки]
        B4 -->|Нет| B6[Зафиксировать успех]
        B5 --> B7
        B6 --> B7
    end
    
    Step2 --> Step3
    
    subgraph Step3["ШАГ 2: Запись результатов в память"]
        direction TB
        C1[Добавить в completed_tasks] --> C2[Обновить current_task статус]
        C2 --> C3[Сохранить: summary, файлы, артефакты]
        C3 --> C4[Добавить событие в history]
        C4 --> C5[💾 Записать в project.json]
    end
    
    Step3 --> Step4
    
    subgraph Step4["ШАГ 3: Директор читает контекст"]
        direction TB
        D1[Прочитать project.json] --> D2[Загрузить task_history]
        D2 --> D3[Загрузить completed_tasks]
        D3 --> D4[Загрузить current_task]
        D4 --> D5[Анализ полного контекста]
    end
    
    Step4 --> Decision
    
    subgraph Decision["❓ Принятие решения"]
        direction TB
        E1{Задача выполнена полностью?} -->|НЕТ| E2[Продолжить работу]
        E1 -->|ДА| E3{Все артефакты на месте?}
        E3 -->|НЕТ| E2
        E3 -->|ДА| E4{Нет критических ошибок?}
        E4 -->|НЕТ| E2
        E4 -->|ДА| E5[✅ ЗАВЕРШИТЬ]
        E2 --> E6[Сформировать новую подзадачу]
    end
    
    Decision --> Final
    
    E5 --> Final1([🏁 ФИНИШ: Задача завершена])
    E6 --> Step1
    
    style Start fill:#4CAF50,color:#fff
    style Final1 fill:#2196F3,color:#fff
    style Step1 fill:#E3F2FD
    style Step2 fill:#FFF3E0
    style Step3 fill:#E3F2FD
    style Step4 fill:#F3E5F5
    style Decision fill:#FFEBEE
```

## Ключевые точки контроля

| Точка | Что проверяем | Где записано |
|-------|--------------|--------------|
| **A** | Задача получена и записана | `task_history`, `current_task` |
| **B** | Агенты завершили работу | `completed_tasks` |
| **C** | Результаты сохранены | `project.json` |
| **D** | Директор прочитал контекст | Вся память проекта |
| **E** | Решение: продолжить/завершить | На основе анализа всех данных |

## Пример трассировки одной задачи

```
1. 📥 ЗАДАЧА: "Создай файл README.md"
   ↓
2. 💾 ЗАПИСЬ В ПАМЯТЬ:
   {
     "task_id": "task_001",
     "status": "in_progress",
     "description": "Создай файл README.md",
     "timestamp": "2024-01-15T10:00:00Z"
   }
   ↓
3. 🤖 АГЕНТЫ РАБОТАЮТ:
   - Agent пишет код
   - Создаёт файл README.md
   ↓
4. 💾 ЗАПИСЬ РЕЗУЛЬТАТОВ:
   {
     "task_id": "task_001",
     "status": "completed",
     "summary": "Файл создан успешно",
     "files_created": ["README.md"],
     "artifacts": {...}
   }
   ↓
5. 📖 ДИРЕКТОР ЧИТАЕТ:
   - task_history: [task_001]
   - completed_tasks: [task_001]
   - current_task: {status: "completed"}
   ↓
6. ❓ РЕШЕНИЕ:
   - Задача выполнена? → ДА
   - Файл на месте? → ДА
   - Ошибок нет? → ДА
   ↓
7. 🏁 ФИНИШ: Задача завершена
```

## Правила для директора

### ✅ Продолжать работу если:
- Задача выполнена не полностью
- Есть недостающие артефакты
- Обнаружены критические ошибки
- Нужна дополнительная проверка

### 🏁 Завершать работу если:
- Все требования выполнены
- Все артефакты созданы и проверены
- Нет критических ошибок
- Пользователь получил результат
