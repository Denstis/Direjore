# Алгоритм передачи контекста для Директора

## Проблема
Директор не понимает предыдущий контекст работы агентов, что приводит к:
- Повторению уже выполненных задач
- Потере информации о результатах
- Невозможности принять обоснованное решение о завершении работы

## Решение
Внедрён **алгоритм передачи контекста** с записью всех событий в память проекта.

---

## 📋 Структура памяти проекта (project.json)

```json
{
  "task_history": [
    {
      "id": 1,
      "timestamp": "2025-01-15T10:30:00",
      "task": "Создать модуль аутентификации",
      "status": "assigned",
      "iteration": 0
    }
  ],
  "current_task": {
    "id": 1,
    "task": "Создать модуль аутентификации",
    "assigned_at": "2025-01-15T10:30:00",
    "status": "in_progress",
    "completed_at": null,
    "result_summary": null
  },
  "completed_tasks": [
    {
      "id": 1,
      "timestamp": "2025-01-15T10:45:00",
      "task_id": 1,
      "status": "completed",
      "summary": "Модуль аутентификации создан",
      "files_created": ["src/auth.py"],
      "files_modified": [],
      "errors": [],
      "artifacts": {},
      "recommendations": [],
      "tool_calls_count": 3
    }
  ],
  "history": [
    {
      "timestamp": "2025-01-15T10:45:00",
      "event_type": "agent_completion",
      "role": "coder",
      "success": true,
      "summary": "Модуль аутентификации создан"
    }
  ]
}
```

---

## 🔄 4 этапа алгоритма

### ЭТАП 1: При получении задания → Запись в память
**Когда:** Пользователь отправляет запрос  
**Кто:** Conductor  
**Что записывается:**
- ID задачи
- Текст задания
- Время получения
- Статус: "assigned" → "in_progress"

**Код:**
```python
await self._log_task_assignment(user_message)
```

---

### ЭТАП 2: После выполнения агента → Запись результатов
**Когда:** Агент завершил работу (успешно или с ошибкой)  
**Кто:** Conductor  
**Что записывается:**
- ID завершённой задачи
- Статус: "completed" или "failed"
- Краткое описание (summary)
- Список созданных/изменённых файлов
- Ошибки (если есть)
- Артефакты
- Рекомендации

**Код:**
```python
if event.get("type") == "agent_done":
    await self._log_agent_completion(event)
```

---

### ЭТАП 3: Перед принятием решения → Чтение из памяти
**Когда:** Директор выбирает следующее действие  
**Кто:** Conductor (для Директора)  
**Что читается:**
- Вся история задач (task_history)
- Все завершённые задачи (completed_tasks)
- Текущее задание (current_task)
- Общая история событий (history)

**Код:**
```python
memory_context = await self._get_project_context()
task_history = memory_context.get("task_history", [])
completed_tasks = memory_context.get("completed_tasks", [])
```

---

### ЭТАП 4: Принятие решения → Продолжить или Завершить
**Когда:** Директор анализирует результаты  
**Кто:** Директор (на основе контекста)  

**Критерии решения:**

| Ситуация | Решение |
|----------|---------|
| Все задачи выполнены успешно | ✅ FINAL (завершить) |
| Есть ошибки в последних задачах | 🔄 DELEGATE (коррекция) |
| Нужны уточнения у пользователя | ❓ ASK_USER |
| Задачи выполняются частично | 🔄 DELEGATE (следующий шаг) |

**Пример JSON для FINAL:**
```json
{
  "action": "final",
  "result": "Все задачи выполнены успешно",
  "artifacts": {
    "files": ["src/auth.py", "tests/test_auth.py"]
  }
}
```

## 📊 Визуализация потока

```
┌─────────────┐
│ Пользователь │
│  "Сделай X"  │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────┐
│  ЭТАП 1: Запись задания         │
│  ─────────────────────          │
│  • task_history.append()        │
│  • current_task.status =        │
│    "in_progress"                │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  ЭТАП 2: Делегирование агенту   │
│  ─────────────────────          │
│  Worker.execute()               │
│  → выполнение инструментов      │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  ЭТАП 3: Запись результатов     │
│  ─────────────────────          │
│  • completed_tasks.append()     │
│  • current_task.status =        │
│    "completed"/"failed"         │
│  • history.append()             │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  ЭТАП 4: Чтение директором      │
│  ─────────────────────          │
│  • _get_project_context()       │
│  • Анализ task_history          │
│  • Анализ completed_tasks       │
└──────────────┬──────────────────┘
               │
               ▼
         ┌─────────────┐
         │  Решение:   │
         │  YES → DONE │
         │  NO → LOOP  │
         └─────────────┘
```

## ✅ Преимущества алгоритма

1. **Полный контекст** — Директор видит всю историю работы
2. **Отслеживание прогресса** — Видно какие задачи выполнены, какие нет
3. **Обоснованные решения** — Решения принимаются на основе данных
4. **Аудит** — Вся история сохраняется в project.json
5. **Восстановление** — Можно продолжить с любого момента

---

## 🔧 Технические детали

### Методы Conductor

| Метод | Назначение | Вызов |
|-------|-----------|-------|
| `_log_task_assignment()` | Запись задания при получении | Этап 1 |
| `_log_agent_completion()` | Запись результатов агента | Этап 3 |
| `_get_project_context()` | Чтение контекста для директора | Этап 4 |
| `_get_state_context()` | Текущее состояние проекта | Этап 4 |

### Файлы

- **conductor.py** — Логика алгоритма
- **project.json** — Хранилище памяти проекта
- **state.json** — Техническое состояние (стадия, итерации)

## 📝 Пример использования

### Сценарий: Создание веб-приложения

1. **Запрос пользователя:**
   ```
   "Создай простое веб-приложение с Flask"
   ```

2. **Этап 1 — Запись задания:**
   ```json
   {
     "task_history": [{
       "id": 1,
       "task": "Создай простое веб-приложение с Flask",
       "status": "in_progress"
     }]
   }
   ```

3. **Этап 2 — Делегирование coder:**
   ```
   Роль: coder
   Задача: Создать app.py с Flask маршрутами
   ```

4. **Этап 3 — Запись результатов:**
   ```json
   {
     "completed_tasks": [{
       "id": 1,
       "task_id": 1,
       "status": "completed",
       "summary": "Flask приложение создано",
       "files_created": ["app.py"]
     }]
   }
   ```

5. **Этап 4 — Решение директора:**
   - Читает: task_history, completed_tasks
   - Видит: app.py создан, но нет тестов
   - Решение: DELEGATE tester (продолжить)

6. **Повтор этапов 2-4 для tester**

7. **Финальное решение:**
   - Читает: все задачи выполнены
   - Решение: FINAL (завершить)

---

## 🎯 Ключевой принцип

> **ВСЁ ЗАПИСЫВАТЬ** при получении задачи и завершении агентов  
> **ВСЁ ЧИТАТЬ** из памяти перед принятием решения

Это обеспечивает полную прозрачность и позволяет Директору принимать обоснованные решения о продолжении или завершении работы.
