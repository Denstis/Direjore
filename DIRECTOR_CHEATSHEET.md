# Шпаргалка для Директора: Логика работы с контекстом

## 📋 Простое объяснение

**Проблема:** Директор не помнит что было сделано ранее и не может принять решение о завершении работы.

**Решение:** Единая "память проекта" (файл `project.json`), куда записывается ВСЁ:
- Какие задачи получали
- Кто их выполнял
- Что получилось в результате
- Какие были ошибки

---

## 🔄 ЦИКЛ РАБОТЫ (4 шага)

### Шаг 1️⃣: ПОЛУЧИЛ ЗАДАЧУ → ЗАПИШИ
```
Пользователь говорит: "Сделай Х"
↓
Директор записывает в память:
{
  "task_log": [
    {
      "type": "task_received",
      "description": "Сделай Х",
      "timestamp": "2025-01-15T10:00:00"
    }
  ],
  "current_task": {
    "status": "pending",
    "description": "Сделай Х"
  }
}
```

---

### Шаг 2️⃣: ОТДАЛ АГЕНТУ → ЗАПИШИ
```
Директор решает: "Эту задачу сделает coder"
↓
Запись в память:
{
  "task_log": [
    ...,
    {
      "type": "task_delegated",
      "role": "coder",
      "task": "Напиши код для Х"
    }
  ],
  "current_task": {
    "status": "in_progress",
    "assigned_to": "coder"
  }
}
```

---

### Шаг 3️⃣: АГЕНТ СДЕЛАЛ → ЗАПИШИ РЕЗУЛЬТАТ
```
Агент coder возвращается с отчётом:
{
  "status": "success",
  "summary": "Создан файл index.html",
  "files_created": ["index.html"]
}
↓
Директор записывает в память:
{
  "task_log": [
    ...,
    {
      "type": "task_completed",
      "status": "success",
      "summary": "Создан файл index.html",
      "files_created": ["index.html"]
    }
  ],
  "current_task": {
    "status": "completed",
    "result_summary": "Создан файл index.html"
  },
  "artifacts": {
    "index.html": {...}
  }
}
```

---

### Шаг 4️⃣: ПРОВЕРЬ ПАМЯТЬ → ПРИМИ РЕШЕНИЕ
```
Директор читает полную память проекта:
- Смотрит current_task.status → "completed" ✅
- Смотрит errors → пустой список ✅
- Смотрит artifacts → файлы созданы ✅

↓
РЕШЕНИЕ:
┌─────────────────────────────────────┐
│ Если ВСЁ выполнено → FINAL         │
│ Если нужно ещё сделать → DELEGATE  │
│ Если нужна информация → ASK_USER   │
│ Если проблема → CORRECT/ERROR      │
└─────────────────────────────────────┘
```

---

## 🎯 КРИТЕРИИ РЕШЕНИЙ (простая таблица)

| Вопрос | Ответ YES | Ответ NO |
|--------|-----------|----------|
| **Все шаги плана выполнены?** | → FINAL | → Продолжай |
| **Есть критические ошибки?** | → ASK_USER или ERROR | → Продолжай |
| **Нужно уточнение у пользователя?** | → ASK_USER | → Продолжай |
| **Ещё есть работа по плану?** | → DELEGATE | → FINAL |

## 🔍 ПРИМЕР: Как директор принимает решение

### Сценарий А: Задача выполнена
```
📖 Чтение памяти:
  - current_task.status = "completed"
  - errors = []
  - Все шаги плана = выполнены
  
💭 Мысли директора:
  "Всё готово, ошибок нет, пользователь доволен"
  
✅ Решение: action = FINAL
```

### Сценарий Б: Нужно продолжить
```
📖 Чтение памяти:
  - current_task.status = "completed"
  - Но план имеет ещё 3 шага
  - Ошибок нет
  
💭 Мысли директора:
  "Первый шаг сделан, но нужно сделать ещё 3"
  
✅ Решение: action = DELEGATE (следующий шаг)
```

### Сценарий В: Проблема
```
📖 Чтение памяти:
  - current_task.status = "failed"
  - errors = ["Не удалось подключиться к БД"]
  - Agent рекомендации = ["Проверьте настройки"]
  
💭 Мысли директора:
  "Есть ошибка, нужно уточнить у пользователя"
  
✅ Решение: action = ASK_USER
```

## ⚡ БЫСТРАЯ ПРОВЕРКА (чек-лист)

Прежде чем принять решение, проверь:

- [ ] Прочитал `current_task.status`?
- [ ] Посмотрел последние 5 записей в `task_log`?
- [ ] Проверил `errors` на наличие проблем?
- [ ] Увидел все созданные `artifacts`?
- [ ] Знаешь сколько шагов в плане осталось?

**Если всё проверил → можешь принимать обоснованное решение!**

---

## 💡 СОВЕТЫ ДИРЕКТОРУ

1. **Всегда читай память перед решением** - это твой единственный источник истины
2. **Смотри на тренды** - если много ошибок подряд, maybe стоит изменить подход
3. **Архивируй завершённые задачи** - переноси из `current_task` в `completed_tasks`
4. **Записывай свои решения** - добавляй в `decisions` почему выбрал тот или иной путь
5. **Не доверяй последнему событию** - смотри всю историю `task_log`

---

## 🚨 ТИПИЧНЫЕ ОШИБКИ

❌ **Решать без чтения памяти** → потеря контекста
❌ **Игнорировать ошибки** → накапливаются проблемы
❌ **Не записывать результаты** → следующий агент не знает что сделано
❌ **Забывать архивировать** → память разрастается

✅ **Правильный подход:**
```python
# Перед КАЖДЫМ решением
memory = await self._get_project_context()
state = await self._get_state_context()

# Анализ
if memory['current_task']['status'] == 'completed' and not memory['errors']:
    # Можно завершать или продолжать
    ...
```
