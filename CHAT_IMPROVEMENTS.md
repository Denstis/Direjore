# 📋 План улучшений: Отображение событий в чате

## 🎯 Проблема
Пользователь видит только системные сообщения без деталей:
```
👤 Вы: создай тестовый текстовый файл

[20:15:18] 
⚙️ Система: 🤖 Делегировано роли ****
📋 Задача: 

[20:15:36] 
❌ Ошибка: Задача выполнена
```

**Не видно:**
- Какую задачу получил агент
- Какие инструменты использовал
- Что конкретно сделал
- Какие файлы создал/изменил

---

## ✅ Реализованные улучшения

### 1. **Детализация делегирования задачи**
**Файл:** `conductor_app/gui/app.py` (метод `_on_delegated`)

**Было:**
```python
message = f"🤖 Делегировано роли **{role}**\n\n📋 Задача: {task}"
```

**Стало:**
```python
tools = data.get('tools', [])
message = f"🤖 Делегировано роли **{role}**\n\n📋 Задача: {task}"

if tools:
    tools_str = ', '.join(tools)
    message += f"\n\n🔧 Доступные инструменты: {tools_str}"
```

**Пример вывода:**
```
⚙️ Система: 🤖 Делегировано роли coder

📋 Задача: Создать тестовый текстовый файл в проекте

🔧 Доступные инструменты: write_file, list_files
```

---

### 2. **Детализация результатов работы агента**
**Файл:** `conductor_app/gui/app.py` (метод `_on_agent_done`)

**Было:**
```python
summary = report.get('summary', 'Задача выполнена')
message_parts = [summary]
```

**Стало:**
```python
tool_calls = report.get('tool_calls', [])

message_parts = [f"✅ Задача выполнена" if success else f"⚠️ Задача выполнена с ошибками"]

if tool_calls:
    tools_used = [tc.get('tool_name', '') for tc in tool_calls if isinstance(tc, dict)]
    if tools_used:
        message_parts.append(f"\n\n🔧 Использованы инструменты: {', '.join(tools_used)}")

if files_created:
    message_parts.append(f"\n📁 Создано файлов: {files_list}")
    
if files_modified:
    message_parts.append(f"\n✏️ Изменено файлов: {files_list}")
```

**Пример вывода:**
```
🤖 Агент: ✅ Задача выполнена

🔧 Использованы инструменты: write_file

📁 Создано файлов: test.txt
```

---

### 3. **Спойлеры для сворачивания подробностей**
**Файл:** `conductor_app/gui/panels/chat_panel.py`

**Новая функция:**
- Метод `add_message()` теперь принимает параметр `collapsible=True`
- Длинные сообщения можно сворачивать/разворачивать кликом
- Экономит место в чате

**Пример использования:**
```python
self.chat_panel.add_message("assistant", full_message, collapsible=True)
```

**Визуально:**
```
🤖 Агент: ✅ Задача выполнена
  🔧 Использованы инструменты: write_file
  📁 Создано файлов: test.txt
  [Нажмите, чтобы свернуть/развернуть]
```

---

### 4. **Передача списка инструментов из Conductor**
**Файл:** `conductor_app/src/director/conductor.py`

**Изменение:**
```python
yield {
    "type": "delegated",
    "role": action.payload.role,
    "task": action.payload.task,
    "tools": all_allowed_tools,  # ← Добавлено
}
```

Теперь GUI получает полный список инструментов, которые директор выделил агенту.

## 📊 Итоговый пользовательский опыт

### Полный цикл задачи:

**1. Пользователь отправляет запрос:**
```
👤 Вы: создай тестовый текстовый файл
```

**2. Директор делегирует задачу:**
```
⚙️ Система: 🤖 Делегировано роли coder

📋 Задача: Создать тестовый текстовый файл в проекте

🔧 Доступные инструменты: write_file, list_files
```

**3. Агент выполняет задачу (видны вызовы инструментов):**
```
🔧 Инструмент: 🔧 Вызов инструмента write_file

Параметры: path=test.txt, content="Hello World"
```

**4. Агент завершает работу:**
```
🤖 Агент: ✅ Задача выполнена

🔧 Использованы инструменты: write_file

📁 Создано файлов: test.txt
```

**5. Директор подтверждает завершение:**
```
🤖 Агент: 🎉 Готово: Тестовый файл успешно создан
```
