# Алгоритм передачи контекста для Директора

## 🎯 Основная цель

Обеспечить директору полный контекст работы системы при принятии решений о продолжении или завершении задачи.

### Ключевой принцип
> **ВСЁ ЗАПИСЫВАТЬ** при получении задачи и завершении агентов  
> **ВСЁ ЧИТАТЬ** из памяти перед принятием решения о продолжении/завершении

## 🔧 Технические детали реализации

### Метод 1: `_log_task_assignment(user_message)`
**Когда вызывается:** Сразу после получения запроса от пользователя

**Что делает:**
```python
async def _log_task_assignment(self, user_message: str):
    task_id = len(self.project_memory.get("task_history", [])) + 1
    
    task_record = {
        "id": task_id,
        "message": user_message,
        "timestamp": datetime.now().isoformat(),
        "status": "in_progress"
    }
    
    self.project_memory.setdefault("task_history", []).append(task_record)
    self.project_memory["current_task"] = {
        "id": task_id,
        "message": user_message,
        "status": "in_progress",
        "started_at": task_record["timestamp"],
        "iterations": 0
    }
    
    await self._save_project_memory()
    logger.info(f"Задание записано в память проекта под ID {task_id}")
```

---

### Метод 2: `_log_agent_completion(event)`
**Когда вызывается:** После завершения работы агента (успешно или с ошибкой)

**Что делает:**
```python
async def _log_agent_completion(self, event: AgentFinishedEvent):
    completed_task = {
        "task_id": self.project_memory.get("current_task", {}).get("id"),
        "agent_role": event.role,
        "summary": event.summary,
        "status": event.status.value,
        "files_created": event.files_created or [],
        "timestamp": datetime.now().isoformat(),
        "artifacts": {
            "output": event.output,
            "tool_calls": event.tool_calls or []
        }
    }
    
    if event.error:
        completed_task["error"] = event.error
    
    self.project_memory.setdefault("completed_tasks", []).append(completed_task)
    
    if "current_task" in self.project_memory:
        self.project_memory["current_task"]["status"] = (
            "completed" if event.status == AgentStatus.SUCCESS else "failed"
        )
        self.project_memory["current_task"]["iterations"] = \
            self.project_memory["current_task"].get("iterations", 0) + 1
    
    self.project_memory.setdefault("history", []).append({
        "type": "agent_completed",
        "task_id": completed_task["task_id"],
        "agent_role": event.role,
        "status": event.status.value,
        "timestamp": completed_task["timestamp"]
    })
    
    await self._save_project_memory()
    logger.info("Результаты работы агента записаны в память проекта")
```

### Метод 3: `_get_project_context()`
**Когда вызывается:** Перед каждым решением директора (продолжить или завершить)

**Что делает:**
```python
async def _get_project_context(self) -> dict:
    await self._load_project_memory()
    
    return {
        "task_history": self.project_memory.get("task_history", []),
        "completed_tasks": self.project_memory.get("completed_tasks", []),
        "current_task": self.project_memory.get("current_task"),
        "history": self.project_memory.get("history", [])
    }
```

**Как используется в промпте директора:**
```python
context = await self._get_project_context()

context_summary = []
if context["task_history"]:
    context_summary.append(f"Всего задач: {len(context['task_history'])}")
if context["completed_tasks"]:
    context_summary.append(f"Выполнено агентами: {len(context['completed_tasks'])}")
    for task in context["completed_tasks"][-3:]:
        context_summary.append(f"  - {task['agent_role']}: {task['summary']}")

system_prompt += "\n\n📋 КОНТЕКСТ ПРОЕКТА:\n" + "\n".join(context_summary)
```

| 19:52:09 | Агент coder: SUCCESS | ⚠️ | Ошибка записи в память (WinError 183) |
| 19:52:20 | Директор: DELEGATE | ✅ | Роль=coder (итерация 2) |
| 19:52:33 | Агент coder: write_file | ✅ | Файл создан |
| 19:52:54 | Агент coder: SUCCESS | ⚠️ | Ошибка записи в память (WinError 183) |
| 19:53:07 | Директор: DELEGATE | ✅ | Роль=coder (итерация 3) |
| ... | ... | ... | Зацикливание... |

### Проблема:
Ошибка `WinError 183: Невозможно создать файл, так как он уже существует` при переименовании `project.tmp` → `project.json`

### Решение:
Использовать атомарную запись с заменой существующего файла:
```python
def _save_project_memory(self):
    tmp_path = self.memory_file.with_suffix('.tmp')
    with open(tmp_path, 'w', encoding='utf-8') as f:
        json.dump(self.project_memory, f, indent=2, ensure_ascii=False)
    
    # Windows-safe rename: сначала удалить целевой файл
    if self.memory_file.exists():
        self.memory_file.unlink()
    
    # Затем переименовать временный файл
    tmp_path.rename(self.memory_file)
```

## ✅ Чек-лист проверки перед завершением задачи

Директор должен проверить следующие пункты перед принятием решения FINISH:

- [ ] **Задача выполнена полностью?**
  - Все требования пользователя реализованы
  - Нет незавершённых подзадач
  
- [ ] **Результаты записаны в память?**
  - completed_tasks содержит последнюю операцию
  - files_created указан корректно
  - status = "success"
  
- [ ] **Файлы созданы физически?**
  - Файлы существуют в директории проекта
  - Содержимое файлов соответствует требованиям
  
- [ ] **Ошибок нет?**
  - В last_agent_result нет поля error
  - В логах нет критических ошибок
  
- [ ] **Пользователь получил ответ?**
  - Финальный отчёт сформирован
  - Результат возвращён через chat_panel

### Таблица решений директора:

| Условие | Решение | Действие |
|---------|---------|----------|
| Задача выполнена, ошибок нет | **FINISH** | Завершить, вернуть результат |
| Задача не выполнена полностью | **DELEGATE** | Отправить агенту на доработку |
| Есть ошибка в выполнении | **DELEGATE** | Попросить агента исправить |
| Файлы не созданы | **DELEGATE** | Попросить создать файлы |
| Превышено кол-во итераций (10) | **FINISH** | Завершить с сообщением о лимите |

## 🛠️ Типичные проблемы и решения

### Проблема 1: Зацикливание из-за ошибки записи
**Симптом:** Многократное повторение итераций без завершения

**Причина:** Ошибка `_save_project_memory()` не обрабатывается, директор не видит результатов

**Решение:**
- Обработать исключение при записи
- Логировать ошибку, но продолжать выполнение
- Использовать Windows-safe метод переименования файлов

---

### Проблема 2: Директор не видит результаты агентов
**Симптом:** Повторное делегирование той же задачи

**Причина:** `_get_project_context()` не вызывается перед решением

**Решение:**
- Обязательно вызывать `_get_project_context()` перед анализом
- Добавлять контекст в system prompt директора
- Проверять completed_tasks перед DELEGATE

---

### Проблема 3: Агент не может создать файл
**Симптом:** Ошибка "Инструмент не найден в реестре"

**Причина:** Инструмент зарегистрирован под другим именем

**Решение:**
- Проверить реестр инструментов: `ToolRegistry.list_tools()`
- Использовать правильное имя инструмента в промпте
- Синхронизировать имена инструментов между агентом и реестром

## 💡 Рекомендации по использованию

### Для разработчиков:
1. Всегда вызывайте `_log_task_assignment()` сразу после получения запроса
2. Всегда вызывайте `_log_agent_completion()` после завершения агента
3. Всегда вызывайте `_get_project_context()` перед решением директора
4. Обрабатывайте исключения при записи в память проекта
5. Используйте атомарную запись файлов (tmp + rename)

### Для директора (LLM):
1. Проверяй `completed_tasks` перед принятием решения
2. Если задача выполнена → выбирай FINISH
3. Если есть ошибки → выбирай DELEGATE с исправлением
4. Считай итерации в `current_task.iterations`
5. При 10 итерациях → завершай с сообщением о лимите

### Для отладки:
1. Включи логирование уровня INFO для conductor
2. Проверяй content/projects/{id}/memory/project.json
3. Следи за полем `current_task.status`
4. Проверяй `completed_tasks[-1]` на наличие ошибок
